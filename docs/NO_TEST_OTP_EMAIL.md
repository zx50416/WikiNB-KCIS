# 禁止亂發測試驗證信

**不要**對真實同仁信箱（`@kcis.com.tw`、`@kcis.ntpc.edu.tw` 等）呼叫會寄出 OTP／驗證碼的 API。

名單上的人**不一定會註冊**；亂發信會造成困擾與不信任。

## 可以怎麼驗

- 只測**邏輯／網域規則**：讀程式、`domain.js`、roster，或 mock；不要打 `lookup` / `send-code` 到真人信箱
- 需要真寄信時：只用**明確同意的測試信箱**（目前：`chaos60649@gmail.com`、學生測試 `st101031616@gmail.com`），並先問使用者；勿對 roster 裡其他同仁發信
- 除錯 SMTP：用測試信箱或本機 `DEV_LOG_CODE`，不要拿同事當收件人

## 一句話

沒有使用者同意 → 不發驗證信。
